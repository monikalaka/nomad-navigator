[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's look at the system architecture. 
First, when a traveler sends a request, the START node routes it straight to the Security Checkpoint function node. 
This checkpoint automatically scans the query for any sensitive personal information, filters out prompt injections, and blocks restricted destinations. 
If the check is clean, the query is routed to the travel_orchestrator LlmAgent. 
The travel_orchestrator then delegates tasks to our two specialized sub-agents: the itinerary_planner agent to draft a day-by-day plan, and the logistics_coordinator agent to verify transportation and local advice. 
Both sub-agents connect directly to our local MCP Server, utilizing tools like get_attractions, get_weather_forecast, calculate_transit_time, and get_local_tips to ground their planning. 
Finally, the flow converges on a Human Review Gate, which pauses execution to seek traveler confirmation before sending the final travel guide to the final_output node.
